## 一、系统安装与启动

### 1.1 前置要求

在开始前，请确认本机已安装以下软件：

| 软件 | 最低版本 | 下载地址 |
|---|---|---|
| Python | 3.8 | https://www.python.org/downloads/ |
| Node.js | 18 | https://nodejs.org/ |
| MySQL | 5.7（推荐 8.0） | https://dev.mysql.com/downloads/ |
| Git | 任意 | https://git-scm.com/ |

---

### 1.2 第一步：获取代码

```bash
git clone https://github.com/LDan777/DatabaseProject.git
cd DatabaseProject
```
或者下载作业上传压缩包。

---

### 1.3 第二步：初始化数据库

**方式 A — 命令行：**

```bash
mysql -u root -p < airline_ticket.sql
```

**方式 B — 图形客户端（Navicat / DBeaver）：**

1. 新建连接，连接本地 MySQL（host: 127.0.0.1，port: 3306）
2. 右键"运行 SQL 文件"，选择项目根目录下的 `airline_ticket.sql`
3. 执行完成后应看到名为 `airline_ticket` 的数据库，包含以下表：

```
passenger · city · airport · flight · flight_stop
flight_instance · cabin_price · ticket_record · vip_discount
```

---

### 1.4 第三步：启动后端

```bash
cd ds-aviation-backend
pip install flask flask-cors pymysql
python app.py
```

启动时终端会提示：

```
请输入你的本地 MySQL 数据库密码:
```

输入 MySQL 的 root 密码（输入时不显示字符，直接回车确认）。

启动成功后看到：

```
 * Running on http://0.0.0.0:5000
 * Debug mode: on
```

> **提示：** 后端需要保持运行，请勿关闭这个终端窗口。

---

### 1.5 第四步：启动前端

另开一个终端窗口：

```bash
cd airline-frontend
npm install
npm run dev
```

启动成功后看到：

```
  VITE v8.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
```

---

### 1.6 访问系统

| 角色 | 地址 |
|---|---|
| 乘客（购票端） | http://localhost:5173/ |
| 管理员（后台） | http://localhost:5173/admin |

## 二、乘客端操作指南

### 2.1 注册账号

1. 打开 http://localhost:5173/
2. 点击页面上的 **「注册」** 按钮
3. 填写以下信息：
   - **身份证号**：18位，作为登录账号（唯一标识）
   - **姓名**：真实姓名
   - **手机号**：11位手机号
   - **密码**：自定义登录密码
4. 点击 **「提交注册」**
5. 注册成功后自动跳转登录界面

> 每个身份证号只能注册一次，再次注册会提示"该身份证已注册"。

---

### 2.2 登录

1. 在登录界面输入 **身份证号** 和 **密码**
2. 点击 **「登录」**
3. 登录成功后进入乘客主界面

登录后界面右上角显示当前用户姓名与会员等级（普通 / 银卡 / 金卡）。

---

### 2.3 查询航班

1. 在主界面的搜索区域填写：
   - **出发城市**：输入城市名或机场名（支持模糊搜索，如"北京"、"首都"）
   - **到达城市**：同上
   - **出行日期**：点击日期选择器，选择出行日期
2. 点击 **「查询」**

搜索结果展示：

| 字段 | 说明 |
|---|---|
| 航班号 | 如 CA1001 |
| 起降城市/机场 | 北京首都 → 上海虹桥 |
| 起降时间 | 08:00 → 10:05 |
| 当前季节 | 旺季 / 平季 / 淡季（影响价格） |
| 头等舱 | 余票数 + 实际售价 |
| 经济舱 | 余票数 + 实际售价 |

> 实际售价 = 标准价 × 季节系数 × 会员折扣（含提前购票折扣）。

---

### 2.4 购买机票

1. 在搜索结果中找到目标航班
2. 点击 **「购买头等舱」** 或 **「购买经济舱」**
3. 弹出确认框，显示航班信息和最终价格
4. 点击 **「确认购买」**
5. 购票成功后可在"我的行程"中查看订单

> 同一身份证不能重复购买同一航班同一日期的票。  
> 余票为 0 时购票按钮置灰，无法点击。

---

### 2.5 查看我的行程

1. 点击顶部导航栏的 **「我的行程」** 标签
2. 页面列出所有历史订单，按出行日期降序排列

每条订单显示：

| 字段 | 说明 |
|---|---|
| 订单号 | ticket_id |
| 航班信息 | 航班号、起降城市、起降时间 |
| 出行日期 | 具体飞行日期 |
| 舱位 | 头等舱 / 经济舱 |
| 实付价格 | 实际支付金额 |
| 订单状态 | 已支付 / 已退票 / 已改签 / 已完成 |

---

### 2.6 退票

1. 在"我的行程"中找到需要退票的订单
2. 订单状态为 **「已支付」** 时才显示退票按钮
3. 点击 **「退票」**
4. 确认弹框 → 点击 **「确认退票」**
5. 成功后订单状态变为 **「已退票」**，座位自动归还

> 已完成、已退票、已改签的订单不可再退票。

---

### 2.7 改签

1. 在"我的行程"中找到需要改签的订单
2. 点击 **「改签」**
3. 在改签界面重新搜索同路线的其他航班/日期
4. 选择目标航班和舱位（须与原订单舱位一致）
5. 点击 **「确认改签」**
6. 成功后：
   - 原订单状态变为 **「已改签」**
   - 新订单自动生成，状态为 **「已支付」**

> 改签不支持跨城市（起落城市须一致）。  
> 目标航班无余票时无法改签。

---

### 2.8 系统设置与个人信息

1. 点击顶部导航的 **「系统设置」** 或个人头像
2. 展示当前账户信息：
   - 姓名
   - 身份证号
   - 联系电话（已绑定的手机号）
   - 会员等级

> 会员等级由管理员在后台修改，乘客本人不可自行更改。

## 三、管理员端操作指南

访问地址：http://localhost:5173/admin

> 当前版本管理员页面无需单独登录，直接访问即可。

管理页左侧为导航菜单，包含以下模块：

---

### 3.1 城市管理

**路径：管理后台 → 城市管理**

**查看城市列表：**
- 页面加载后自动显示所有城市记录（区号、城市名、省份）

**新增城市：**
1. 点击 **「新增城市」**
2. 填写：
   - 区号（唯一，如 010）
   - 城市名（如 北京）
   - 所属省份（如 北京市）
3. 点击 **「保存」**

**修改城市：**
1. 点击目标行的 **「编辑」**
2. 修改城市名或省份（区号不可更改）
3. 点击 **「保存」**

**删除城市：**
1. 点击目标行的 **「删除」**
2. 确认后删除（若该城市下有关联机场，需先删除机场）

---

### 3.2 机场管理

**路径：管理后台 → 机场管理**

**新增机场：**
1. 点击 **「新增机场」**
2. 填写：
   - 机场代码（IATA 三字码，如 PEK）
   - 机场名称（如 首都国际机场）
   - 所属城市（从城市列表选择）
3. 点击 **「保存」**

**修改 / 删除：** 与城市管理操作方式相同。

---

### 3.3 航班航线管理

**路径：管理后台 → 航班管理**

**新增航班：**
1. 点击 **「新增航班」**
2. 填写航班基本信息：
   - 航班号（如 CA1001）
   - 机型（如 B737）
   - 头等舱座位数
   - 经济舱座位数
   - 执飞周期（勾选周几执飞，如"周一至周五"）
   - 起飞时间 / 降落时间
3. 填写经停信息（至少填起飞机场和降落机场）：
   - 顺序选择：起飞 / 经停1 / 经停2 / 降落
   - 每一条选对应机场
4. 填写舱位定价：
   - 头等舱标准价（元）
   - 经济舱标准价（元）
5. 点击 **「保存」**

> 新增航班后，系统自动为当日起**两个月**按执飞周期生成航班实例。

**修改航班：**
1. 点击目标航班的 **「编辑」**
2. 修改相关字段
3. 点击 **「保存」**

**删除航班：**
- 删除航班的同时，会删除该航班所有关联的航班实例

---

### 3.4 航班实例管理

**路径：管理后台 → 航班实例管理**

航班实例代表某航班在特定日期的一次飞行。

**手动新增实例：**
1. 点击 **「新增实例」**
2. 选择已有航班号
3. 输入日期（YYYY-MM-DD）
4. 保存后，系统自动从航班表取座位数和时刻填充实例数据

**修改实例（调整状态/余票/实际时间）：**
1. 在列表中找到目标实例
2. 点击 **「编辑」**
3. 可修改：
   - 航班状态（计划 / 延误 / 取消）
   - 头等舱余票
   - 经济舱余票
   - 实际起飞时间
   - 实际降落时间
4. 点击 **「保存」**

**删除实例：**
- 删除指定日期的航班实例（不影响航班航线数据）

---

### 3.5 乘客管理

**路径：管理后台 → 乘客管理**

**查看乘客列表：**
- 显示所有注册乘客的身份证、姓名、手机号、会员等级

**修改会员等级：**
1. 点击目标乘客的 **「修改等级」**
2. 选择新等级（普通 / 银卡 / 金卡）
3. 保存

> 会员等级影响购票折扣，规则在 `vip_discount` 表中配置。

---

### 3.6 订单流水查看

**路径：管理后台 → 订单管理**

- 展示全部订单流水，含乘客信息、航班信息、支付金额、订单状态
- 支持按状态筛选（已支付 / 已退票 / 已改签 / 已完成）
- 管理员仅可查看，不可在此界面直接修改订单（退票改签由乘客在 C 端操作）
