# ✈️ 航空票务管理系统 - 前端项目

> 一个完整的 B2C 在线航班订票平台，包括客户端购票系统（C端）和管理后台（B端）。
> 本项目为数据库课程设计配套前端系统，为航空票售票及运维提供可视化操作平台，支持与符合 3NF 设计的后端数据库进行全量联调。

- **本地联调模式：** 现已全面转为本地开发模式以支持后端 API 集成。前端需配合本地/云端运行的后端服务使用。

### 1. 环境准备
- 安装 [Node.js](https://nodejs.org/) (修建议 v16+) 及 Git。

## 🔌 API 核心功能模块

前端 `src/api/` 下的三个模块已与后端数据库结构（用户、航班实例、售票记录等表）深度绑定：

### 1. 认证模块 (`auth.js` - 7个接口)

* 手机号验证码发送与用户注册/登录
* 乘客身份证绑定（关联所有的购票与实名行为）
* 获取/更新个人信息及个人档案管理

### 2. 航班模块 (`flight.js` - 16个接口)

* **乘客智能航班搜索：** 支持出发地、目的地、日期过滤及空状态处理。
* **后台大盘控制：** 管理员对航班信息、航线、起降时间、基础票价等进行 CRUD（增删改查）。
* **舱位实例与定价：** 单航班多舱位（头等/经济）配额监控、淡旺季调价与会员折扣计算。

### 3. 订单模块 (`order.js` - 11个接口)

* **在线机票预订：** 触发后端高并发事务，支持 `SELECT ... FOR UPDATE` 锁机制防止机票超卖。
* **行程记录与流转：** 乘客个人中心查看历史订单；支持自主“一键退票”（自动回收舱位配额）。
* **后台运维队列：** 全量订单流水、待处理的挂起/人工处理异常订单队列。

## ⚙️ 前后端联调配置指南

### 1. 修改后端服务地址

如果你和后端同学在本地或者云端跑起来了服务，请打开 `src/api/request.js` 修改 `baseURL`：

```javascript
const service = axios.create({
  baseURL: 'http://localhost:8080/api',  // ← 改为你们实际的后端运行地址（例如 Node.js 的 3000 端口）
  timeout: 5000
});

```

### 2. 跨域与认证

* **CORS配置：** 由于前端运行在 `5173` 端口，后端运行在其他端口，请务必在后端配置 **CORS 允许跨域请求**。
* **Token 携带：** 前端在请求拦截器中已配置自动挂载：`Authorization: Bearer <token>`。
